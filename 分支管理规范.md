# 分支管理规范

### 环境分支
- master 发布环境 
- prod  产环境
- dev   开发环境
> 以上三个分支没有提交代码,合并分支,解决冲突,确认运行无误
> 相关人员拉去代码的分支

### 权限管理
> 线上gitee/github/gitLab 中建立分组.只有组员才能提交代码,拉取代码
### 工作分配
#### 组长
- 1.创建规范 开发规范,提交规范,测试规范
- 2.创建初始化工程,并提交到dev
- 3.负责维护服务端代码,并读懂在开发中服务端遇到的问题
- 4.负责每个人的工作任务分工
- 5.负责 持续集成过程中,审核prod中过程,prod中代码为npm run build
- 6.负责督促在规定时间内完成任务,或者提前完成任务.任务不能完成组长负责

#### 组员
- 1.根据组长指定规范 安排任务完成代码编写,拉取dev中代码进行开发
- 2.有问题大大方方问组长,如果因为你个人拖慢进度,请全班吃饭
- 3.按照规范持续集成,配合组长测试,学习 合并处理以及团队合作,主动配合积极完成任务
